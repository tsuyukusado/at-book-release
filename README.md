# VSCodeとは

VSCode（Visual Studio Code）は、Microsoftが無償で提供する高機能テキストエディタです。

もともとはプログラマー向けに作られたものですが、その強力なカスタマイズ性から、今では小説執筆にも活用されています。

「プログラマーのためのツールでしょ？」と思われたかもしれません。でも、ちょっと待ってください。

VSCodeには、書き手にとってうれしい特徴があります。

- **無料**で使える
- Windows・macOS・Linuxすべてで動作する
- スマホからも執筆できる（GithubへのPushが必要）
- gitによるバージョン管理で、原稿の変更履歴を自動で記録できる
- 加筆修正が思うように行かなかった時も、戻りたいデータにいつでも巻き戻せる

こう書かれても、いまいちピンと来ていないかもしれません。

# ＠本とは

＠本（アットぼん）は、日本語の執筆から組版までを一直線に繋いでくれる、新たな執筆記法です。

縦書き・ルビ・目次・表紙テンプレート……同人誌や自費出版に必要なものが、シンプルな記法で全部まかなえます。

難しい記法は必要ありません。全ては、**＠マーク**です。

## ＠本の記法

＠本の記法はシンプルです。

- ＠大見出し

　大見出しができます。

- ＠＠小見出し

　小見出しが出来ます。

- ＠目次

　目次が自動生成されます。

- ＠＠＠

　改ページされます。

- ＠漢字（ルビ）

　ルビが振られます。

- ＠文字（・）

　圏点が打たれます。

- ・・

　三点リーダーに変換されます。

- ーー

　ダッシュに変換されます。

以上が全てです。

気が付きましたか？

これらの記法は全て、**JISキーボードの日本語入力モードで完結する**のです。

これが、日本人のための新しい日本語記法。＠本です。

# 実際の導入方法

## 1. Node.js のインストール

＠本の動作には **Node.js v18 以上** が必要です。

### macOS

```sh
brew install node
```

Homebrew が入っていない場合は、[Node.js 公式サイト](https://nodejs.org/ja) から LTS 版をダウンロードしてインストールしてください。

### Windows

[Node.js 公式サイト](https://nodejs.org/ja) から LTS 版のインストーラーをダウンロードして実行してください。

インストール中のオプションはすべてデフォルトのままで構いません。

### Linux (Ubuntu/Debian)

```sh
sudo apt install nodejs npm
```

インストール後、以下のコマンドでバージョンを確認してください。`v18` 以上であれば問題ありません。

```sh
node -v
```

> **TeX Live は不要になりました。** ＠本の組版エンジンは HTML/CSS ベースに刷新され、PDF 生成には [Vivliostyle](https://vivliostyle.org/) を利用します。LaTeX（LuaLaTeX / TeX Live）のインストールは必要ありません。

## 2. ＠本 のインストール

```sh
git clone https://github.com/tsuyukusado/at-book-release.git
cd at-book-release
npm install
```

`npm install` を実行すると、自動的にビルドと git フックのセットアップ、そして PDF 組版に必要な依存（Vivliostyle）の導入が行われます。

なお、**初めて PDF を生成するとき**に、組版用のヘッドレスブラウザ（Chromium）が自動でダウンロードされます（数百 MB／初回のみ）。ネットワークに接続した状態で最初の生成を行ってください。

これで、`.atb` ファイルをコミットするたびに PDF が自動生成される環境が整いました。

## 3. VS Code 拡張のインストール（任意）

`.atb` ファイルの構文強調表示を有効にします。VSCodeで執筆する場合はぜひ入れておいてください。

1. VS Code の左サイドバーで拡張機能アイコンをクリック（または `Ctrl+Shift+X` / `Cmd+Shift+X`）
2. 右上の `…` メニューから「VSIX からインストール...」を選択
3. `vscode-atb/vscode-atb-0.1.0.vsix` を選択してインストール
4. VS Code を再起動

# 使い方

## 基本

- docというフォルダに、拡張子を`.atb`にしたファイルをつくります。
- 自由に小説を書きます。
- gitにコミットします。

基本の使い方はこれだけです。

## 生成されたPDFの場所

自動で生成されるPDFは、distというディレクトリに保存されています。

生成結果が気になる時は、随時確認しましょう。

## PDF生成の設定を変更する

PDF生成時の、紙の大きさと縦書きか横書きかは変更することができます。

`at-book.config.json` を編集してください。

```json
{
  "paperSize": "a6",
  "writingMode": "vertical"
}
```

| プロパティ | 値 | デフォルト |
|---|---|---|
| `paperSize` | `a4` / `a5` / `a6` / `b5` | `a6` |
| `writingMode` | `vertical`（縦書き）/ `horizontal`（横書き）| `vertical` |

## EPUB（電子書籍）を生成する

紙のPDFだけでなく、**EPUB（電子書籍）** も生成できます。設定ファイルに `formats` を追加してください。

```json
{
  "paperSize": "a6",
  "writingMode": "vertical",
  "formats": ["pdf", "epub"]
}
```

| プロパティ | 値 | デフォルト |
|---|---|---|
| `formats` | `pdf` / `epub` の配列 | `["pdf"]`（未指定時） |

- `["pdf", "epub"]` … PDFとEPUBの両方を生成します。
- `["epub"]` … EPUBのみを生成します。
- 未指定（項目なし）… 従来どおりPDFのみを生成します。

生成されたEPUBは `dist/<ファイル名>.epub` に保存されます。ルビ・圏点・縦書きはそのまま引き継がれます。

> **EPUBはリフロー型です。** 読者が端末で文字サイズを変えられる形式のため、ページ番号（ノンブル）・綴じ代・コロフォンといった「紙面固定の装飾」や表紙テンプレートは付きません（これらはPDF専用です）。また本文フォントは埋め込まず、読者の端末のフォントで表示されます。

## 表紙テンプレートを生成する

＠本では、生成したPDFのページ数に応じた**表紙のテンプレート**も生成することが出来ます。

上述した設定ファイルに、以下の項目を追加してください。

```json
{
  "bodyPaperThicknessMm": 0.09,
  "coverPaperThicknessMm": 0.35
}
```

| プロパティ | 値 |
|---|---|
| `bodyPaperThicknessMm` | 本文用紙の厚さ（mm）例: `0.09` |
| `coverPaperThicknessMm` | 表紙用紙の厚さ（mm）例: `0.35` |

両方設定した状態でコミットすると、本文PDFと同時に `dist/sample-hyoshi.svg`（背幅・塗り足しガイド入り）が自動生成されます。

## 手動でPDF生成を実行する

上述した全ての自動生成機能は、コマンドを使って手動で生成する事も可能です。

### 本文PDFと表紙テンプレートをまとめて生成する場合

```sh
at-book doc/sample.atb
# → dist/sample-honbun.pdf が生成される
```

### 表紙テンプレートをページ数指定で単体生成する場合

```sh
at-book cover 160 0.09 0.35
# → dist/cover-template.svg が生成される
```

引数は順に「ページ数」「本文用紙の厚さ（mm）」「表紙用紙の厚さ（mm）」です。

## 文字数・ページ数を記録する

執筆の進み具合を、コミット履歴に沿って記録できます。

```sh
at-book count
```

- **初回**は、これまでの**すべてのコミット**をさかのぼって、各コミット時点の文字数を `char-count.log` に書き出します。
- **2回目以降**は、前回からの**新しいコミットだけ**を処理します。
- 各コミットには、**前回コミットからの文字数の増減**が `(前回比 +123文字)` の形で付きます（そのファイルが初めて登場したコミットは `(新規)`）。

`char-count.log` の中身はこんな形です。

```
コミット: a1b2c3d 第三章を書き上げた
ファイル: doc/sample.atb
総文字数: 12,345文字 (前回比 +1,200文字)
```

ページ数は、実際に組版してみないと分からない値です。そのため履歴には残さず、**PDFを生成したとき**（`at-book doc/sample.atb` やコミット時の自動生成）に、ページ数と「前回生成時からの増減」を一緒に記録します。

特定のファイルだけを今すぐ数えたいときは、ファイルを渡してください。

```sh
at-book count doc/sample.atb
```
